# 资讯电台工作规划

## 产品定义

Guo News Radio 是一档由 AI 主持的个人资讯节目。它要解决的不是“哪里有新闻”，而是“今天什么信息值得占用我的注意力，以及它们之间有什么关系”。

## MVP 边界

第一版只做一件事：生成每日一期 8 到 12 分钟的中文资讯节目。

暂不做：

- 多人协作。
- 完整账号系统。
- 复杂推荐算法。
- 全平台音箱投放。
- 商业化后台。

先做：

- 关注域配置。
- 资讯源配置。
- 候选资讯结构化。
- 价值排序。
- 事件聚类。
- 节目脚本生成。
- 前端播放原型。

## 任务拆解

### 0. 产品地基

- 定义电台人格：冷静、判断型、少废话、能解释为什么重要。
- 定义用户注意力规则：哪些领域值得听，哪些来源降权，哪些内容直接过滤。
- 定义一期节目的结构：开场、今日主线、三到五条重点、一个深挖、结尾追踪。

### 1. 数据输入

- 建立 `sources.json`。
- 支持 RSS 源。
- 支持手动粘贴 URL。
- 支持 newsletter 文本导入。
- 支持 YouTube transcript 作为后续能力。

### 2. 资讯标准化

每条资讯统一成：

```json
{
  "id": "string",
  "title": "string",
  "url": "string",
  "source": "string",
  "published_at": "datetime",
  "summary": "string",
  "entities": ["company", "person", "topic"],
  "domain": "ai/business/tech/world/china",
  "signals": {
    "novelty": 0,
    "impact": 0,
    "personal_relevance": 0,
    "source_quality": 0,
    "follow_up_value": 0
  }
}
```

### 3. 价值排序

排序不是“热度优先”，而是五个问题：

1. 这件事新在哪里？
2. 它改变了谁的约束条件？
3. 它和我的长期关注有什么关系？
4. 它是事实、观点还是营销噪音？
5. 它值得后续追踪吗？

### 4. 节目编排

节目不逐条播报，而是按叙事组织：

- 今日主线：今天最值得占用注意力的结构变化。
- 快讯组：三条短、硬、新的信息。
- 深挖组：一条值得解释背景和影响的信息。
- 观察清单：需要继续追踪的人、公司、政策、论文或产品。

## 里程碑

### M1：可听节目生成

- 手动准备 10 条资讯样本。
- 生成一期完整节目脚本。
- 前端原型可展示章节和播放状态。

### M2：半自动抓取

- RSS 抓取。
- 去重。
- 评分。
- 生成 `episode.json`。

### M3：真实播放

- 接入 TTS。
- 生成 mp3。
- 支持章节跳转。

### M4：长期记忆

- 事件链追踪。
- 实体画像。
- 来源可信度动态调整。
- 用户口味更新。
